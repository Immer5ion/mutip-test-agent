## 5. 场景优先级规则

每个场景按以下维度评分：

- 影响度（1-3）：失败后的业务影响。
- 发生概率（1-3）：基于复杂度、变更频率与依赖风险。
- 可发现性（1-3）：失败是否容易在早期被发现。

优先级建议：

- P0：总分 >= 7，或直接影响收入/安全/合规。
- P1：总分 5-6。
- P2：总分 <= 4。

## 7. 用例草案

### 用例：TC-RG-P0-001 无凭证访问受保护接口

- 目标风险：未授权控制失效
- 前置条件：已登录任意控制台页面
- 数据准备：无
- 操作步骤：
1. 在页面上下文调用 `fetch('/console/api/workspaces', { credentials: 'omit' })`
2. 记录状态码
- 预期结果：返回 `401`
- 清理步骤：无
- 备注：已执行，`noAuthStatus = 401`

### 用例：TC-RG-P0-002 API 基础 URL 配置正确性

- 目标风险：外部接入失败
- 前置条件：进入 `/app/{id}/develop`
- 数据准备：无
- 操作步骤：
1. 查看 API 服务器基础 URL
2. 比对部署对外地址
- 预期结果：基础 URL 应为可访问域名/网关，不应为 `127.0.0.1`
- 清理步骤：无
- 备注：本次仍失败（缺陷持续）

### 用例：TC-RG-P0-003 标注空提交必填校验

- 目标风险：无效标注数据入库
- 前置条件：进入 `/annotations`
- 数据准备：无
- 操作步骤：
1. 点击“添加标注”
2. 不输入提问/回复直接提交
- 预期结果：提示“提问不能为空”
- 清理步骤：取消弹窗
- 备注：通过

### 用例：TC-RG-P0-004 用户名空值不可保存

- 目标风险：账户主数据不完整
- 前置条件：进入 `/account`
- 数据准备：无
- 操作步骤：
1. 打开用户名编辑弹窗
2. 清空输入框
3. 尝试点击保存
- 预期结果：保存按钮禁用，点击不可达
- 清理步骤：取消弹窗
- 备注：通过（点击超时因按钮 disabled）

### 用例：TC-RG-P0-005 密码不一致校验

- 目标风险：错误密码被提交
- 前置条件：进入 `/account`
- 数据准备：原密码、新密码、确认密码（不一致）
- 操作步骤：
1. 打开重置密码
2. 输入不一致的新密码与确认密码
3. 提交
- 预期结果：提示“两次密码不相同”
- 清理步骤：取消弹窗
- 备注：通过

### 用例：TC-RG-P1-001 导航连续性（深链接+刷新+前后退）

- 目标风险：复杂路由场景状态丢失

- 备注：通过

### 用例：TC-RG-P1-002 工具搜索双分支

### 用例：TC-RG-P1-003 外部知识库连接参数联动
